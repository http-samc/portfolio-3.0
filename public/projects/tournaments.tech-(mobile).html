What is it?

tournaments.tech is the largest and most accurate debate ranking system. We made this resource because there were no holistic rankings for the public forum national circuit, and existing record curations fail to collect sufficient quality data. Our speedy API delivers you the results from the latest national circuit tournaments just minutes after they occur, and you can stay up to date with instant push notifications. You can use our automated bid list (with ghost bid support) to check your progress on qualifying for the Tournament of Champions. Or, just run a quick search to get intel on your next opponent!

Get the tournaments.tech app on the Play Store or the App Store, or access it anywhere at tournaments.tech!

What I learned?

I made this app with React Native. I already wrote the API to get the data from tabroom (available at http-samc/tabroom-API) and have endpoints set up for the web version of the app (read more at @smrth/tournaments.tech). This app centered around tying everything together into a mobile package. I maintained most of the styling from the website, and a lot of my work focused on lower level things like fetching raw json. Unlike my other mobile app projects that have more specific use cases, I built this system to scale extremely large, so I didn't want to use Firebase üî• because the fees stack up quickly. So, I had to make a lot of calls to my own API that pulls from my mongodb database.

You can check out the source code on GitHub at http-samc/tournaments.tech-mobile.

Functionality

  • Pulls from the Web API (so they're always in sync)
  • Provides browser modal to access issue/feedback reporting
  • All functional views are native screens
  • Scaleable design (phones and tablets)
  • Push notifications to alert users of added tournaments (provided by native notify)
  • Search view by team code or name(s)
  • Built in Bid List (rendered from markdown)
  • About page that details when the dataset was last updated
  • Custom leaderboard table component with pagination
    • uses display height to calculate the optimal amount of rows
    • adds in dummy rows to prevent table from getting smaller on the last page (if ENTRIES % ROWS doesn't equal 0)
  • Swipe down to refresh Bid List and Leaderboard
